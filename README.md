 🏨 AlertX – Smart Emergency Response System for Hospitality

**AlertX** is a real-time emergency response and incident management system tailored for the **hospitality industry** — including hotels, resorts, and guest facilities.

It helps ensure **guest safety, rapid staff coordination, and efficient incident handling** through instant alerts, live tracking, and role-based control.

---

## 🌟 Why AlertX for Hospitality?

In hotels and resorts, emergencies like **medical issues, fire hazards, security threats, or guest distress** require **immediate action**.

AlertX enables:

* Faster response times
* Better communication between staff
* Improved guest safety experience

---

## 🔥 Key Features

* 🚨 **Guest SOS System**
  Guests can instantly trigger alerts using a simple SOS button.

* 🆔 **Unique Guest ID System**
  Each guest is assigned a random ID (e.g., `GUEST-839201`) for tracking without exposing personal data.

* 🛎️ **Live Incident Dashboard**
  Staff can monitor all active incidents in real-time (`incidents.html`).

* 👨‍💼 **Role-Based Access**

  * **Admin (Hotel Manager)** → Full control + analytics
  * **Staff (Security / Reception / Housekeeping)** → Respond to alerts
  * **Guest** → Raise SOS alerts

* 📍 **Location Awareness**
  Tracks guest location (GPS or zone like Room / Floor / Block).

* 🔔 **Instant Notifications**
  Alerts staff immediately when a guest triggers SOS.

* 🧠 **AI Assistant (Extendable)**
  Provides guidance for emergency situations (future AI integration).

* 📊 **Admin Analytics Dashboard**

  * Incident counts
  * Emergency type tracking (Fire / Medical / Security)
  * Staff response monitoring

* 🎤 **Voice SOS (Ready for Integration)**
  Guests can trigger alerts using voice commands (future-ready feature).

* ⚡ **Offline Support (LocalStorage)**
  Works without backend for demo; can be upgraded to cloud.

## 🛠️ Tech Stack

* **Frontend:** HTML, Tailwind CSS, JavaScript
* **Storage:** LocalStorage (demo mode)
* **APIs:**

  * Geolocation API
  * Browser Notification API
  * Web Audio API

---

## 🏗️ How It Works

1. Guest logs in → gets a **unique Guest ID**
2. Guest triggers SOS (Fire / Medical / Security)
3. Alert is stored and broadcasted instantly
4. Staff views it in **Live Incidents Dashboard**
5. Staff updates status:

   * Active → Pending → Resolved
